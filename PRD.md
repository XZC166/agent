# 产品需求文档 (PRD) - 仿真场景自动生成智能体系统

## 1. 项目背景
本项目旨在开发一个基于单智能体（Single Agent）和检索增强生成（RAG）技术的系统，用于自动生成仿真场景（特别是 OpenFOAM/BlastFoam 相关的仿真案例）。系统利用现有的 OpenFOAM 教程描述作为知识库，根据用户的自然语言需求，自动规划并生成相应的仿真配置文件和目录结构。

## 2. 项目目标
- **自动化生成**：通过 LLM 理解用户需求，自动生成复杂的仿真配置文件。
- **知识复用**：利用 RAG 技术检索相似的历史案例（Tutorials），提高生成的准确性和规范性。
- **端到端执行**：智能体不仅生成代码/文本，还能直接在文件系统中创建文件和文件夹，完成场景搭建。

## 3. 用户流程
1. **输入**：用户提供自然语言描述的需求（例如：“建立一个3D建筑物爆炸模拟...”）以及输出的目标路径。
2. **检索**：系统通过 RAG 模块在 `rag_database/openfoam_tutorials` 中检索最相关的案例描述。
3. **规划与生成**：智能体结合用户需求和检索到的参考案例，生成执行计划。
4. **执行**：智能体按照计划在指定的目标路径下创建文件夹、写入配置文件。

## 4. 功能需求

### 4.1 RAG 模块
- **知识库构建**：能够读取 `rag_database/openfoam_tutorials` 下的 Markdown 文件，并进行向量化存储。
- **检索功能**：根据用户输入的 Query，计算相似度，返回 Top-1 个最相关的案例内容。
- **实现细节**：目前基于 `rag_database/embedder.py`，使用 OpenAI 接口进行 Embedding。

### 4.2 智能体 (Agent) 模块
- **核心逻辑**：接收“用户需求” + “参考案例”作为 Context。
- **能力**：
    - **思考 (Reasoning)**：分析需求，拆解步骤（如：先创建 0 文件夹，再创建 constant，system 等）。
    - **工具使用 (Tool Use)**：具备文件系统操作能力（创建目录、写文件）。
- **实现位置**：`agents/blastfoamagent.py` (待开发)。
- **框架**：使用 **LangChain** 框架开发。

### 4.3 交互接口
- **输入方式**：纯命令行工具 (CLI)，支持多轮对话。
- **输出反馈**：实时显示智能体的思考过程和执行动作。
- **验证**：生成结束后，自动执行语法检查和试运行（如 `checkMesh`），确保案例可运行。
