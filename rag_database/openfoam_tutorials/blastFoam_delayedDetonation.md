--- OpenFOAM Tutorial Case Description: Delayed Detonation ---

## Case Overview
**Case Name:** delayedDetonation
**Solver:** blastFoam
**OpenFOAM Version:** Not specified

## Physics Description
This case simulates the programmed, multi-point, delayed detonation of a cylindrical explosive charge in an axisymmetric domain. The explosive (RDX) is initiated at several points along its length at different times, creating a complex, interacting blast wave. The simulation uses adaptive mesh refinement to capture the multiple shock fronts.

## Geometry
- **Type:** 2D axisymmetric wedge
- **Dimensions:** A wedge with a radius of 0.5m and a length of 1.0m.
- **Description:** A 10-degree wedge is used to represent a cylindrical domain, reducing the problem to 2D.

## Mesh Configuration
- **Type:** Structured mesh with adaptive refinement.
- **Initial Cells:** 25 x 50 x 1
- **Mesh Generator:** blockMesh
- **Adaptive Refinement:** Enabled via `dynamicMeshDict`, based on density gradient, with a maximum refinement level of 2.

## Boundary Conditions

### Velocity (U)
- **walls:** `slip`
- **outlet:** `zeroGradient`
- **wedge0, wedge1:** `wedge`

### Pressure (p)
- **walls:** `zeroGradient`
- **outlet:** `pressureWaveTransmissive`
- **wedge0, wedge1:** `wedge`

### Other Fields (T, alpha, etc.)
- **walls, outlet:** `zeroGradient`
- **wedge0, wedge1:** `wedge`

## Initial Conditions
- **U (velocity):** uniform (0 0 0) m/s
- **p (pressure):** uniform 101298 Pa
- **T (temperature):** uniform 300 K
- **alpha.RDX (RDX volume fraction):** Initialized to 1 in a cylindrical region (radius 0.05m, length from y=0.01 to y=0.5) using `setFieldsDict`. `alpha.gas` is set to 0 in this region.

## Time Control
- **Start Time:** 0 s
- **End Time:** 50e-6 s (50 microseconds)
- **Delta T:** 1e-8 s (adaptive time stepping is on)
- **Write Interval:** 1e-6 s

## Numerical Schemes
- **Time derivative:** Euler with `RK2SSP` integrator.
- **Flux Scheme:** HLLC
- **Gradient:** `cellMDLimited leastSquares 1`
- **Interpolation:** `vanLeer` limiter for reconstructed fields.

## Solution Methods
- **Solvers:** `diagonal` for most fields (explicit), `PBiCGStab` for `U` and `e`.

## Physical Properties
- **Phases:** `RDX` (detonating) and `gas` (air).
- **RDX:**
  - Type: `detonating` with a `programmedIgnition` activation model.
  - Reactants EOS: `Murnaghan`.
  - Products EOS: `JWL`.
- **Air:** `idealGas` equation of state.

## Key Features
- **Solver:** `blastFoam`.
- **Programmed Detonation:** Demonstrates the `programmedIgnition` model, where detonation is initiated at specified points with specific time delays and radii. This allows for the simulation of complex initiation sequences.
- **Multi-Point Initiation:** The charge is detonated at 6 different points with varying delays, creating a non-ideal explosion scenario.
- **Adaptive Mesh Refinement (AMR):** Used to capture the multiple, interacting shock waves generated by the complex initiation.
- **Axisymmetric Simulation:** Efficiently models a 3D cylindrical problem in 2D.

## Use Cases
- Simulating complex explosive initiation systems.
- Studying the interaction and focusing of shock waves from multiple detonation points.
- Validating advanced initiation models in blast simulation software.
- A tutorial for the `programmedIgnition` feature in `blastFoam`.
